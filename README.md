# LLM Cases

LLM Cases 是一套面向编码模型的可复现能力测试库。每个 Case 是一个完整的真实场景（如"做一只桌面宠物"），模型在隔离的 Claude Code 会话里按 README 指令完成实现，结果可直接横向比较。

## 设计原则

- **场景为本。** 每个 Case 是一个完整的真实场景，不是某项能力的切片。能力点通过 Case 内部结构观察，而不是预先按能力分类。
- **README 即指令。** 每个 Case 目录的 `README.md` 是 AI 直接读的指令文档（不存在"给人看的部分"和"给 AI 看的部分"的区分），包含场景、需求、约束、评分标准。AI 读完即可开始实现。
- **实例隔离。** 每次跑 Case 在该 Case 目录下创建 `instance1/`、`instance2/` 等递增编号目录，所有交付物放在对应实例目录里，互不污染。
- **单次 vs 多阶段。** 多数 Case 单次交付；少量实操型 Case（如 Case-001 网页迭代）多阶段，镜像真实开发流程。

## 如何运行一个 Case

每个 Case 的标准流程：

1. **读 README**：打开目标 Case 目录下的 `README.md`，里面就是 AI 要读的指令
2. **确认实例目录**：检查该 Case 目录下是否已有 `instance1/`。没有则用 `instance1/`，已有则用 `instance2/`、`instance3/` …
3. **AI 实现**：启动一个全新的 Claude Code 会话，让模型读 README + 实例目录，在实例目录里完成实现
4. **AI 验收**：另起一个全新的 Claude Code 会话（避免上下文污染），让验收模型按 README 的评分标准评价实例产物

整个流程不依赖任何 CLI 或 runner——直接读 README、用 Claude Code 跑、打开浏览器看 `index.html` 即可。

## 实例编号规则

每个 Case 目录下的实例按数字递增，每次新跑用下一个编号：

```text
Case-002-一颗弹珠撞倒一堆多米诺/
├── README.md
├── instance1/    # 第一次跑（如模型 A 的实现）
├── instance2/    # 第二次跑（如模型 B 的实现）
└── instance3/    # 第三次跑
```

- 实例目录是本地运行产物，**不进 git**（`.gitignore` 已忽略 `Case-*/**/*instance*/`）
- 不同实例是独立实现，可横向比较模型差异
- 不要把实例放到 Case 根目录（会污染设计稿）

## 目录结构

```text
llm-cases/
├── README.md              # 本文档：项目说明 + 如何运行 + 实例规则
├── cases总览.md           # 能力分类 + Case 速览
├── .gitignore             # 忽略 instance/、.claude/、.playwright-mcp/ 等
└── Case-NNN-名称/
    ├── README.md          # Case 指令文档（AI 直接读，含场景/需求/约束/评分）
    └── instance1/         # 本地实例产物（不入库）
        ├── index.html
        ├── style.css
        ├── main.js
        └── evidence/
```

## 环境要求

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code)——启动测试和验收会话
- 现代浏览器（Chrome / Edge）——打开实例目录的 `index.html` 看效果
- 不需要 Node.js、PowerShell 脚本、Chrome DevTools Protocol 等额外工具

## 新增 Case

1. 按 `Case-NNN-名称/` 规则创建新目录（NNN 为下一个三位序号）
2. 编写 `README.md`，作为 AI 直接读的指令文档，必须包含：
   - **你的任务**（开场一句，明确告诉 AI 要做什么）
   - **场景**（要做什么的简述）
   - **工作目录与实例编号规则**（从任一现有 Case 复制粘贴即可）
   - **必含元素**（高层清单）
   - **技术约束**（关键技术决策，如物理引擎、状态机等）
   - **不需要做的**（明确边界）
   - **动画原理 / 物理核心**（如果是演示型 Case）
   - **场景设置**（物件规格、风格方向）
   - **动画流程**（多个阶段，标出视觉核心高潮）
   - **视觉与渲染要求**（分项）
   - **交互控制**（具体交互列表）
   - **性能要求 / 代码要求**（约束）
   - **交付标准**
   - **评分标准**（满分 100）
3. 在 `cases总览.md` 的"当前 Case"表里加一行
4. 在本 README 的"当前 Case"表里加一行

每个 Case 的 README 是它唯一的源文档——拿到 README 就能让 AI 跑，不用再翻别处。
