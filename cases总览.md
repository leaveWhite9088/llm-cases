# cases总览

每个 Case 是一个完整的"场景"——模型按 Case 目录下的 README 指令完成实现。
当前版本聚焦**前端能力**，每个 Case 都是综合的真实任务，能力点通过 Case 内部结构（场景设置、动画阶段、视觉核心等）观察，而不是按能力切分 Case。

## 能力分类

| 一级能力   | 二级能力                                         |
| ---------- | ------------------------------------------------ |
| 编码能力   | 前端、后端、架构、调试、测试、部署、运维         |
| Agent 能力 | 规划、工具调用、记忆（长短期）、自我修正、长任务 |
| 知识能力   | 世界知识、垂域知识                               |
| 推理能力   | 逻辑、因果、概率、数学                           |
| 内容能力   | 写作、创意、设计、审美、视频策划                 |
| 多模态能力 | 图片理解、空间理解、3D 场景构建、界面理解        |
| 信息能力   | 检索、归纳、事实核验、来源比较                   |

### Case-001 网页迭代（多阶段）
- 目录：[./Case-001-网页迭代/](./Case-001-网页迭代/)
- 主轴：从零搭出 Slow Brew 咖啡店官网第一版 → 在其上迭代「预约」模块 → 响应式与可访问性打磨
- 三个阶段在同一 workspace 内推进，镜像真实开发流程
- 观察：需求转化、组件结构、视觉层级、代码扩展、布局系统、工程质量

### Case-002 一颗弹珠撞倒一堆多米诺（单次）
- 目录：[./Case-002-一颗弹珠撞倒一堆多米诺/](./Case-002-一颗弹珠撞倒一堆多米诺/)
- 主轴：一颗弹珠撞倒至少 30 张多米诺骨牌触发完整连锁反应，可玩版本含起点调整、排布切换、慢放、音效
- **硬性要求：必须使用 matter.js（或等价 2D 物理引擎），不要自实现碰撞与物理求解**
- 观察：物理引擎、画面感、节奏编排、交互设计、声音同步

### Case-003 一只桌面宠物（单次）
- 目录：[./Case-003-一只桌面宠物/](./Case-003-一只桌面宠物/)
- 主轴：一只住在浏览器角落的像素小猫，能互动、能记住状态、能跨标签页同步
- 三个视觉核心瞬间：开屏冲击 / 被摸的瞬间 / 跨标签页的瞬间
- 观察：角色艺术、动画基础、状态机、浏览器 API、跨页面通信
