### 前端能力

| 案例   | 主要任务               | 核心观察点                           |
| ------ | ---------------------- | ------------------------------------ |
| Case 1 | 从零实现一个响应式页面 | 需求转化、组件结构、基本视觉和完整性 |

| Case 4 | 完成接口与复杂状态交互 | 异步流程、状态管理、异常和边界情况   |

#### Case 1 从零实现一个响应式页面

- 目录：[`1-1-1-从零实现一个响应式页面`](./1-1-1-从零实现一个响应式页面/)
- 状态：已设计，可通过根目录 `case-cli.ps1` 单独、批量或全量执行
- 任务：在空目录中用原生 HTML/CSS/JavaScript 完成 Northstar 响应式产品首页
- 评分：需求覆盖 25、响应式 20、视觉层级 20、工程质量 15、交互 10、可访问性 10
- 隔离：测试和验收可独立执行；每个 Case、每次验收都使用新的 Claude Code 一次性会话

#### Case 2 根据截图还原页面

- 目录：[`1-1-2-根据截图还原页面`](./1-1-2-根据截图还原页面/)
- 状态：已设计，可通过根目录 `case-cli.ps1` 单独、批量或全量执行
- 任务：仅提供一张 1440 × 1000 桌面端参考截图，使用原生 HTML/CSS/JavaScript 还原页面
- 评分：桌面视觉还原 50、响应式推断 20、内容结构 10、细节质感 10、工程质量 5、可访问性与交互 5
- 防作弊：参考图作为只读测试输入并单独快照；禁止将截图直接作为页面图片或背景
- 隔离：测试和验收分别使用新的 Claude Code 会话，验收比较原始参考图与交付截图

#### Case 3 在现有项目修复 Bug

- 目录：[`1-1-3-在现有项目修复Bug`](./1-1-3-在现有项目修复Bug/)
- 状态：已设计，提供故障项目和独立回归测试
- 任务：修复 Pulseboard 的搜索、筛选、排序、持久化和移动导航缺陷
- 评分：功能修复 35、移动交互 20、定位与范围 15、回归健壮性 15、代码质量 10、可访问性与视觉保持 5

#### Case 4 完成接口与复杂状态交互

- 目录：[`1-1-4-完成接口与复杂状态交互`](./1-1-4-完成接口与复杂状态交互/)
- 状态：已设计，提供固定 Mock API 和状态回归测试
- 任务：交付 Relay 订单异常运营台，处理防抖、竞态、分页、详情、选择、乐观更新、失败回滚与重试
- 评分：数据与竞态 20、复杂状态 25、产品功能 20、响应式 15、视觉交互 10、工程与可访问性 10

#### Case 5 综合交付真实功能

- 目录：[`1-1-5-综合交付真实功能`](./1-1-5-综合交付真实功能/)
- 状态：已设计，覆盖完整产品交付闭环
- 任务：从零交付 Signal Loom 客户证据工作台，包含 CRUD、组合筛选、持久化、导入导出和撤销
- 评分：核心闭环 30、数据可靠性 20、复杂状态 15、响应式 15、视觉 10、工程与可访问性 10
