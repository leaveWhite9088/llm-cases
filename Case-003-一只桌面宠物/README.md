# Case-003 一只桌面宠物

> 一只住在浏览器角落的小动物，从会动到有性格，一次交付

## 场景

在浏览器角落养一只像素风格的小动物（默认形象：一只圆滚滚的小猫）。整个 Case 是一次性交付——模型需要在一个工作会话里把整只「能互动、能记住状态、能跨页面跟着你」的宠物做完。

## 需求

### 必含元素

1. **角色视觉**：圆滚滚的像素小猫，暖色调，自包含资源（不依赖外部图片）
2. **基础动作**：待机、走路、转鼠标（看向鼠标方向）至少三个
3. **状态系统**：饥饿、困倦、心情等状态，状态影响动画表现（饿了走路变慢、困了打哈欠、被摸会开心）
4. **互动**：点击 / 拖动有反馈
5. **持久化**：状态保存到 localStorage，下次打开恢复
6. **多标签页同步**：用 BroadcastChannel 或 storage 事件，多标签页是同一只猫

### 技术约束

- 角色资源自包含：纯 CSS / SVG / Canvas 实现，不依赖外部图片文件
- 嵌入方式：Web Component 或浮窗页面（能嵌入任意页面）
- 在 Chrome / Edge 最新版流畅运行
- 渲染方式任选 CSS / SVG / Canvas，但要保证可爱感

### 不需要做的

- 真实桌面应用（Electron 等），浏览器内即可
- 复杂养成系统（数值、等级），简单状态即可
- 移动端适配，桌面浏览器跑通即可
