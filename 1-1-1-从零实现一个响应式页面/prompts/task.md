# 前端实现任务：Northstar 项目协作首页

你正在一个空目录中工作。请从零实现一个名为 **Northstar** 的响应式产品首页。Northstar 是一个帮助小型产品团队规划、追踪和交付工作的协作工具。

## 交付约束

- 只使用原生 HTML、CSS 和 JavaScript，不使用框架、包管理器、CDN、网络字体或任何远程资源。
- 页面必须能直接通过 `index.html` 运行；所有资源保存在当前工作目录内。
- 至少创建 `index.html`、`styles.css` 和 `script.js`。
- 不要把任务说明、设计解释或评分规则写进网页。完成后自行检查文件和页面。
- 禁止使用 base64 图片或复制现成网站。允许使用 CSS 图形和内联 SVG 图标。

## 页面内容

### 1. 顶部导航

- 左侧品牌标志和文字 `Northstar`。
- 桌面端导航项：`Product`、`Solutions`、`Resources`、`Pricing`。
- 右侧有 `Sign in` 和主要按钮 `Start free`。
- 移动端折叠为菜单按钮；点击后可打开和关闭导航。
- 导航容器使用 `id="site-header"`，菜单按钮使用 `id="menu-toggle"`。

### 2. Hero 区

- 小标签文案：`Built for focused teams`。
- 主标题：`Turn ambitious plans into shipped work.`
- 一段简洁的价值说明。
- 两个行动按钮：`Start building — free`、`See how it works`。
- 显示一行社会证明：`Trusted by 2,000+ product teams`。
- Hero 右侧或下方制作一个原创的产品界面预览，不使用外部截图。预览需要包含：项目进度、3 个任务卡片、成员头像或状态、一个小型进度/趋势可视化。
- Hero 容器使用 `id="hero"`，产品预览使用 `id="product-preview"`。

### 3. 信任与数据

- 一组纯文字客户标志：`Arc`、`Layers`、`Quill`、`Sora`、`Vertex`。
- 三个数据指标：`42%`、`3.2×`、`8.5 hrs`，并为每个指标提供解释。

### 4. 功能区

- 标题清楚表达“从计划到交付”的价值。
- 三张功能卡片，主题分别为：`Plan with clarity`、`Stay in sync`、`Learn and improve`。
- 每张卡片有一个轻量的原创视觉元素或图标，不能只有文字。
- 功能卡片容器使用 `id="feature-grid"`。

### 5. 工作视图区

- 展示一个任务列表或看板视图，至少有 4 条内容。
- 提供 `All`、`In progress`、`Completed` 三个筛选按钮，点击后能真实过滤可见项目。
- 筛选容器使用 `id="work-filter"`。

### 6. 最终 CTA 与页脚

- CTA 标题：`Your best work is waiting.`，并包含一个主要按钮。
- 页脚包含品牌、简短说明，以及 Product、Company、Legal 三列链接。
- 显示当前年份或 `2026` 和版权信息。

## 视觉方向

- 气质：现代、克制、可信赖，但不能像未经设计的默认模板。
- 使用暖白或浅中性色作为页面底色，以深墨色作为文字色，并选择一种鲜明但不过度的主色。
- 需要有清楚的空间节奏、版心、字号层级、卡片层级和一致的圆角系统。
- 可以使用细网格、柔和渐变或抽象几何图形增强品牌感，但不能影响可读性。
- 所有可点击元素都应有 hover/focus 状态。

## 响应式要求

- 至少适配 1440px 桌面宽度、768px 平板宽度和 390px 手机宽度。
- 手机端不得出现横向滚动；正文和按钮不能被截断。
- Hero 在桌面端采用双栏布局，在窄屏下变为单栏。
- 功能卡片桌面端三列，平板可两列或合理重排，手机端单列。
- 手机端的主要行动按钮应易于触控，导航菜单必须可用。
- 支持 `prefers-reduced-motion`，避免给偏好减少动效的用户播放非必要动画。

## 可访问性与质量要求

- 使用语义化 HTML，并保持正确的标题层级。
- 菜单按钮需要可访问名称和正确的 `aria-expanded` 状态。
- 筛选按钮需要表达当前选中状态。
- 键盘用户必须能看到清晰的焦点样式。
- 文本与背景需要有足够对比度。
- 浏览器控制台不应出现 JavaScript 错误。
