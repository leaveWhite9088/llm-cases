# 前端调试任务：修复 Pulseboard 问题看板

你接手了一个已有的原生前端项目。请先阅读现有代码，复现并定位问题，然后在原项目内完成修复。不要重写成另一套页面，也不要更换技术栈。

## 项目约束

- 保持原生 HTML、CSS 和 JavaScript，不安装依赖、不使用 CDN 或远程资源。
- 页面入口是 `index.html`；已有视觉风格、内容和文件职责应尽量保留。
- `logic.js` 中的纯函数是回归测试入口，必须保持现有导出函数名称。
- 可以修改已有文件并增加少量必要文件，但不要删除现有业务内容或用硬编码结果绕过逻辑。
- 修复完成后主动运行 `node --check logic.js` 和必要的手工检查。

## 缺陷报告

### PB-101 搜索结果不可靠

- 搜索应忽略用户输入首尾空格和大小写。
- 应同时匹配问题标题与负责人姓名。
- 空搜索词应显示当前状态下的全部问题。

### PB-102 状态筛选错误

- 点击 `All` 应显示全部问题，现在会得到空列表。
- `Open`、`In progress`、`Resolved` 应只显示对应状态。
- 当前筛选按钮必须正确更新 `aria-pressed`。

### PB-103 优先级排序错误且污染数据

- 排序顺序必须是 High → Medium → Low。
- 排序函数不得原地修改传入数组，否则切换筛选后列表顺序会漂移。

### PB-104 已保存状态会导致页面崩溃

- localStorage 为空、JSON 损坏或字段非法时，页面都应使用默认状态启动。
- 合法状态只接受已知筛选值和字符串搜索词。

### PB-105 移动导航和布局不可用

- 390px 宽度下不能出现横向滚动。
- 菜单按钮应打开/关闭侧栏，更新 `aria-expanded`，点击遮罩或按 Escape 可以关闭。
- 从移动端切回桌面宽度后，侧栏和页面状态必须恢复正常。

### PB-106 回归风险

- 顶部统计数字应基于完整数据集，而不是当前筛选结果。
- 搜索与状态筛选必须能够组合使用。
- 找不到结果时展示明确空状态，不能留下空白区域。
- 浏览器控制台不得出现 JavaScript 错误。
