## 物理核心

**必须使用一个成熟的 2D 物理引擎。** 这是本任务最关键的技术决策——本演示的核心挑战是"看起来对"的物理表现，一个稳定、参数可调的物理引擎能解决 90% 的工作；自实现碰撞检测、速度积分、约束求解（手写重力 + AABB）会把任务难度拉满且几乎必然失败。

- 推荐：**matter.js**（轻量、稳定、文档丰富、社区活跃，本任务首选）
- 也可以：planck.js（Box2D 的 JS 移植）、p2.js 等其他等价 2D 物理引擎

本演示的物理表现依赖几个关键常识，物理引擎参数要围绕这些常识调对：

1. **能量传递链**：钢珠的动能 → 第一张牌的势能 → 第二张牌的动能 → ... → 第 N 张牌的势能。每一次传递都有能量损失（碰撞的非完全弹性、骨牌倒下时重心抬升消耗的能量、地面摩擦等），所以连锁会自然衰减——末端牌倒下的速度明显比首张慢，每张牌倒下损失约 10-30% 能量。如果看不到这种衰减，说明物理参数有问题（弹性太高或摩擦太低），链条会"无穷无尽"地传下去。

2. **骨牌是杠杆**：骨牌倒下时是绕底部边缘旋转的杠杆。底部贴地基本不动，顶部划出一个圆弧，速度比底部快得多——这就是为什么一张细长的牌倒下时能撞倒下一张。当牌间距合适时，倒下牌的顶部弧线正好落在下一张的中上部，能量能有效传递；间距过大时，牌倒下但碰不到下一张；间距过小时，牌还没倒下就被击中，能量被"吞掉"。

3. **间隔决定节奏**：骨牌间距是关键变量。
   - 间距 < 牌高 1/2：上一张还没倒下就被击倒，能量被吞掉，连锁看起来"卡住"
   - 间距 > 牌高 2/3：上一张倒下时已经够不到下一张，连锁中断
   - 间距 = 牌高 1/2 到 2/3：连锁完整、节奏自然

4. **摩擦的双重作用**：
   - **钢珠与斜坡**：滚动阻力（约为滑动摩擦的 1/100 到 1/1000），让钢珠能保持速度。如果斜面太光滑，钢珠会打滑而不是滚（看着像溜冰）；如果太粗糙，钢珠滚得很慢，到底部时动能不足。
   - **骨牌与地面**：决定骨牌是"倒下"还是"滑出去"——摩擦太低骨牌会水平滑走而不是自然倒下；摩擦太高骨牌站得稳但倒下需要更大冲击。两者之间要找到平衡。
   - 这两组摩擦系数需要分别调对，互相不矛盾。

5. **钢珠必须重于骨牌**：钢珠撞到第一张牌时，动能要足够大到把牌撞倒且不被反弹回来。钢珠的质量至少是单张骨牌的 5-10 倍，否则会出现"钢珠自己跳开、骨牌只晃了晃"的尴尬场面。

## 场景设置

### 环境氛围

环境氛围是观感的第一来源，有几种合理方向，每个方向都有自己的视觉调性。任选一个方向并保持整场视觉统一。

**方向 A：极简抽象**
纯色背景（浅灰、奶白、米黄等中性色），一束柔和顶光在地面投下柔和阴影，主体骨牌与钢珠占据视觉中心。整体观感像产品概念图——剥离一切非必要元素，让观众专注看物理本身。色板克制（2-3 个主色 + 黑/白/灰）。

**方向 B：桌面玩具感**
浅木纹桌面作为地面，自然光从一侧照入（模拟窗户光），骨牌投下长长的方向性阴影。整体观感像你在桌面上真摆了一排多米诺骨牌，准备推倒它。背景可以有一点景深模糊，让视线集中在前景的骨牌上。

**方向 C：工业展示感**
深色背景（深灰、墨绿、午夜蓝），强侧光从一侧打来，骨牌和钢珠的轮廓被侧光勾出明显的高光与阴影，地面反射感强（金属板或抛光石材）。整体观感像高端工业产品的宣传片。

其他合理方向也可（自然场景、夜景、复古风等），但必须整场视觉统一，不要中途切风格。

### 斜坡
倾斜的平面，角度约 20°-35°（太陡钢珠会跳，太缓滚得慢），钢珠从顶端释放。

斜面需要：
- **合适的摩擦**：与钢珠之间既不打滑也不滚得太慢。判断标准：钢珠能保持滚动（高光在表面移动），到底部时仍有足够动能把第一张牌撞倒。
- **底端平滑过渡**：避免钢珠在斜面-地面交界处跳起（可以用圆弧过渡或短直线斜接）。如果钢珠接触地面那一刻跳起来，整段演示就毁了。
- **材质**：金属（视觉冷、反射强）、木质（视觉暖、哑光）、塑料（中性），任选。材质决定斜面在画面上的视觉表现力，与整体氛围一致即可。

### 骨牌
经典的长方体骨牌，宽厚比约 2:1:4（宽:厚:高，假设总高度 4cm 则宽 2cm 厚 1cm）。单张牌的尺寸要足够看清细节（建议渲染时总高度 50-100 像素），但也不要太大（连锁 30 张总长可能 1-2 米，太长画面装不下）。

**至少 30 张组成完整连锁**，排成一条直线。骨牌之间的间距是本演示的关键参数：

- 间距太小（< 牌高 1/2）：上一张还没倒下就被击倒，能量被吞掉，连锁看起来"卡住"
- 间距太大（> 牌高 2/3）：上一张倒下时已经够不到下一张，连锁中断
- 合适间距（牌高 1/2 到 2/3）：连锁完整、节奏自然

骨牌要有体积感（有厚度、有重量感，不是纸片）。表面材质：白底黑点（经典）、纯色、彩色、木纹等任选。倒下后能看出骨牌的厚度感（不是完全贴合地面的薄片，而是有厚度、有阴影、看起来真的"摔下去"）。

骨牌的密度与摩擦系数要和钢珠质量协调（参考物理核心第 5 点）。

### 钢珠
一个小球，直径约为骨牌宽度的 1/2（太大撞不到精准位置，太小看着轻）。材质：

- **金属感**：高反射、视觉冷、有重量感。表面有明显的高光，滚动时高光位置会变。
- **玻璃质**：半透明、内部有高光，视觉轻盈但仍有重量。适合"透明钢珠"这种奇幻感。
- **塑料中性**：颜色鲜明、视觉活泼。适合儿童玩具感的方向。

任选一种。

钢珠在斜坡上滚动时自身旋转——这是"看起来有重量"的关键。不只是平移，要能看到钢珠表面纹理或高光在滚动过程中的位置变化。如果钢珠看起来像在"飘"，整个动画就垮了。

钢珠的密度应大于骨牌（钢珠质量 >> 骨牌质量），这样钢珠撞到第一张牌时动能足够、不会被反弹回来。

### 地面
骨牌站立的平面。材质：木质、瓷砖、桌面、石板等任选，要与整体氛围一致。

地面需要：
- **能接收阴影**：骨牌和钢珠在地面投出阴影，阴影随物体运动实时更新。没有阴影，骨牌会像贴图飘在地面上。
- **摩擦系数合适**：参考物理核心第 4 点。
- **视觉上不抢戏**：地面是背景，骨牌和钢珠是主体。地面材质要简洁（纯色、淡纹理），不要复杂图案抢了视觉。

## 动画流程

### 阶段 1：释放与下落（2–3 秒）
用户点击「开始」后，钢珠出现在斜坡顶端，从静止释放，在重力作用下沿斜坡向下加速滚动。

钢珠的滚动要有重量感——起始慢、加速、中段最快，不是匀速滑动。具体表现：

- **起始阶段**：钢珠几乎不动，重力分量刚开始克服静摩擦
- **加速阶段**：钢珠速度快速上升，可以看到表面纹理或高光在旋转
- **中段**：钢珠接近匀速（滚动阻力大致平衡重力分量）
- **接近底部**：钢珠仍有足够动能在撞到第一张牌时把它撞倒

画面可以跟随钢珠，或先远景后切近景，给观众"追踪滚下来"的代入感。

### 阶段 2：首次撞击（视觉核心之一，0.3–0.5 秒）
钢珠以一定速度撞上第一张牌，第一张牌倒下击中第二张。

**关键细节：** 首次撞击必须有力学冲击感（不是轻轻碰一下）。这是整个演示的"启动信号"——观众从这里开始期待连锁。

- **撞击瞬间**：可以加粒子效果（灰尘、火花任选其一，或都不要）来强化冲击
- **画面反应**：可以给一个轻微的"撞击反应"（微抖）让观众感受到力度
- **第一张牌的倒下**：要有力——不是轻飘飘的倒下，而是带着整个链条即将开始崩的预感。视觉上：撞击点的骨牌底边附近应该有明显的形变或位移，紧接着它绕底边旋转倒下
- **击打第二张牌**：第一张牌的顶部弧线扫到第二张牌，第二张牌被"推倒"开始连锁

### 阶段 3：连锁反应（视觉核心之二，5–15 秒）
骨牌逐张倒下，能量沿链条传递。这是整个演示最壮观的阶段。

**关键细节：** 节奏感是核心。骨牌间隔有变化——紧密段（间隔小、倒下快速）+ 稀疏段（间隔大、倒下有停顿），整体不要匀速。具体表现：

- **紧密段**：骨牌密集排列，倒下速度极快，几乎是连续的"咔咔咔"声，观众的视线跟不上单个牌
- **稀疏段**：骨牌间距稍大，每张牌倒下后有几毫秒的停顿，下一张再倒下，形成可听的节奏感
- **能量衰减**：末端牌倒下的速度应明显比首张慢（每张牌倒下损失 10-30% 能量），形成自然的衰减感。如果链条末端还在以同样速度倒下，说明物理参数不对
- **画面跟随**：要紧跟"倒下波前"（最新的那张正在倒的牌），让观众的眼睛跟着连锁走
- **音效**：每张牌倒下时有音效（节奏感比音调变化更重要）——可以用单一音高 + 触发时机来制造节奏
- **细节强化**：倒下过程中牌与地面碰撞产生灰尘或反光变化，强化能量释放感

连锁不能中途断（间距要合理）；连锁不能"全段匀速"（要有节奏变化）；末端的牌倒下要有"最后一击"的感觉。

### 阶段 4：尾声（1–3 秒）
所有骨牌倒下，场景归于平静。

最后几张牌的倒下可以给一点时间停留，让观众感受到"最后一击"。画面可以从连锁全景切到一片倒下的骨牌特写——所有骨牌横躺，形成最终的视觉构图。重置按钮让场景回到初始状态。

## 视觉与渲染要求

### 骨牌渲染
有材质感（不是纯色片）。具体表现：

- 表面有轻微反光——不是镜面反射，而是漫反射 + 一点高光
- 允许有纹理（木纹、纸张感等）或纯色（白底黑点是最经典的）
- 骨牌之间的间距视觉清晰——观众的视线能区分相邻两张牌
- 倒下后能看出骨牌的厚度感——不是完全贴合地面的薄片，而是有厚度、有阴影、有"摔下去"的实体感

### 阴影
骨牌和钢珠在地面投出阴影。阴影要：

- 随物体运动实时更新（特别是钢珠滚下斜坡时）
- 阴影方向一致（受同一光源影响）
- 阴影柔和度适中——太硬看起来像贴图，太软看不出立体感

### 明暗
通过明暗对比营造空间感（不需要真实光照计算）。具体表现：

- 骨牌顶面亮一些、侧面暗一些（暗示从上方来的光）
- 钢珠有渐变高光（暗示球面）
- 整体亮度让所有细节清晰可见——不要过暗，否则看不到骨牌的细节

### 画面跟随
跟随"倒下波前"是核心。具体表现：

- 画面始终包含正在倒下的最新那张牌
- 跟随速度可以略快于连锁速度（让观众的视线领先一点）
- 连锁结束时可以从跟随位置平滑切到全景（展示所有倒下的骨牌）
- 不要求电影级运镜，但要有"画面在跟踪事情发生"的感觉

## 性能注意

- Chrome / Edge 最新版流畅运行（目标 60fps，30fps 是底线）
- 至少 30 张骨牌的连锁必须流畅——这是物理引擎 + 渲染管线的综合考验
- 材质应共享——所有骨牌可以共用同一个渲染对象，只是位置/旋转不同（Canvas / SVG 可以考虑实例化或预渲染）
- 不要在动画过程中频繁创建和销毁动态对象——比如每帧 new 一个粒子对象会让 GC 抖动
- 利用物理引擎的 sleep 状态——静止的物体不参与计算，让引擎能高效处理大量物体

## 交互控制

- **「开始」按钮**：触发钢珠释放动画
- **「重置」按钮**：恢复场景到初始状态（钢珠回到顶端、所有骨牌立起）
- **时间缩放滑块**：0.1x – 1.0x 慢动作控制，方便观察连锁细节
- **起点调整**：用户能拖动调整钢珠释放位置（在斜坡上的位置或高度）
