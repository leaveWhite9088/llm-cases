# Case 1-1-2：根据截图还原页面

这个 Case 测试模型能否真正读取界面截图，将视觉信息转化为可运行的前端实现，并从单张桌面截图推断合理的移动端布局。

## 测试设计

- 输入：一张 1440 × 1000 的桌面端产品仪表盘截图。
- 交付：原生 HTML、CSS、JavaScript 页面。
- 桌面证据：1440 × 1000，实现截图应与参考图直接比较。
- 移动证据：390 × 844，不提供移动参考图，用于观察响应式推断能力。
- 防作弊：自动检查会识别把参考截图直接作为图片、背景或 Canvas 贴图的实现。
- 评分重点：桌面视觉还原 50%，响应式推断 20%，其余为结构完整性、细节、工程质量和可访问性。

参考图在测试开始时会复制到待测模型的 `reference/target-desktop.png`。原始文件同时快照到运行结果的 `definition/inputs/`，避免模型修改输入后污染验收证据。

## 运行

只测试：

```powershell
.\case-cli.ps1 test -Cases 1-1-2
```

测试并验收：

```powershell
.\case-cli.ps1 run -Cases 1-1-2 -TestModel "测试模型" -JudgeModel "验收模型"
```

每次测试和每次验收仍使用互相独立的新 Claude Code 会话；结果写入 Cases 同级的 `../../runs`。
