# 前端实现任务：根据截图还原响应式页面

你正在一个仅包含参考素材的工作目录中。请查看 `reference/target-desktop.png`，从零实现截图中的页面。

这是一项视觉还原任务。截图是页面内容、排版、布局、颜色、尺寸、间距、边框、圆角和装饰细节的主要事实来源；不要把它当作灵感图重新设计。桌面端应尽量接近参考截图，移动端则需要根据桌面布局做出合理、完整的响应式推断。

## 交付约束

- 只使用原生 HTML、CSS 和 JavaScript，不使用框架、包管理器、CDN、网络字体或远程资源。
- 页面必须能直接通过 `index.html` 运行，至少创建 `index.html`、`styles.css` 和 `script.js`。
- 所有实现资源保存在当前工作目录内，可使用 CSS 图形和内联 SVG。
- 不得把参考截图本身作为 `<img>`、背景图、Canvas 贴图、base64 数据或其他页面内容来伪装实现。
- 不要在页面中展示任务说明、评分规则、尺寸标注或“参考图”等测试信息。
- 参考截图只提供桌面端版本，宽度为 1440px，高度为 1000px。

## 还原要求

1. 仔细读取参考截图，再开始编码；不要仅凭文件名或这份文字猜测页面。
2. 还原截图中可见的全部主要区域、文字内容、导航状态、数据卡片、图表、列表和装饰细节。
3. 在 1440 × 1000 视口下，页面的整体构图、区域比例、对齐关系、留白和视觉层级应尽量贴近参考图。
4. 在 390 × 844 手机视口下做合理响应式适配：不得横向滚动，内容不能被截断，导航与卡片要自然重排，文字仍应清晰可读。
5. 页面中的按钮、导航项和可交互控件应具有 hover/focus 状态；明显的菜单或筛选控件应能工作。
6. 使用语义化 HTML、正确标题层级、可见键盘焦点，并支持 `prefers-reduced-motion`。
7. 完成后自行检查控制台错误、溢出、资源路径和两个目标视口。

你拥有对当前目录的文件读写和命令执行权限。直接完成交付，不要只给方案或代码片段。
